# Two-note alert candidates

Audio cues with a "two-note alert" shape — pattern recognition rather than
amplitude recognition, on the theory that an interval is faster to perceive
through game ambient than a single transient. Each option is mono,
44.1 kHz, 16-bit PCM WAV.

Listen via `/mnt/station-mods/parry-tell-audio-candidates/alerts/` on the
Windows side (double-click in File Explorer).

## Picking guidance

For a parry-window cue, the priority order is:
1. **Cuts through game ambient.** If you can't hear it during sword clash + boss music, it's useless.
2. **Feels alert-like.** Two notes ascending is the universal "pay attention" signal across UI design.
3. **Short enough to not overlap the next window.** Parryable attacks can come 0.5-1.5s apart; cue should be < 200ms to leave room.
4. **Doesn't sound game-y.** A cue that sounds like a normal Elden Ring SFX will be mentally filtered as ambient.

My picks for first audition (Josh — try these three first):

- **`synth-mgs-ascend-sine-160ms.wav`** — clean, distinctive, no licensing. The MGS reference is doing real work here; humans recognize ascending intervals as "alert" universally.
- **`alert-dslr-double-130ms.wav`** — sharpest of the sampled options. Two notes in 130ms. Very urgent.
- **`alert-beep-up-287ms.wav`** — slightly longer but classic two-note ascending. CC0, no attribution headache.

If none of those land, try the descending variants (`alert-hint-descending`, `synth-descend-third`) — sometimes a descending interval feels less alarming and more informational.

block based on which license that specific file carries (see the table
above). Use exactly ONE of these forms — the right one per file.

alert-eqylizer):

(CC0 doesn't legally require attribution, but it's polite to credit
the original author. The Creative Commons recommended attribution
format is what's shown above.)

as original work (it is — generated by Python script in this session, no
